Detailed Description

These functions allow the operation of a Nüvü Camēras CCCP via a NcGrab handle.

A CCCP controller may be operated without pre-programmed readout modes by using NcGrab as the control context to pass serial commands to the controller and to configure the host system for the reception and processing of images.

Enumeration Type Documentation

enum NcSerialProtocol

Values identifying the serial protocol that can be used to communicate with a Nüvü Camēras device.

The serial protocol will impact the required format of serial commands if the user intends to send them to the camera controller themselves (see ncGrabFormatSerialCommand() for more details).

Enumerator
NC_SERIAL_LIVE 

Use the serial protocol in effect.

NC_SERIAL_MEGANTIC 

Original serial protocol.

NC_SERIAL_PASADENA 

CRC32-based error-checking.

int ncGrabOpen ( int  unit,
int  channel,
int  nbrBuffer,
NcGrab *  grab 
)

Opens a channel from the host system to the camera controller and initializes internal variables.

Initialises a NcGrab object in preparation for controlling the associated camera controller via calls to ncGrab functions: see Controller Functions (NcGrab) for a summary. The user may specify a device via a particular framegrabber unit and channel or use special unit and channel values (see below) to allow automatic discovery of an available device.

The number of loop buffers required to pipeline image acquisition is also set.

Parameters
unitThis parameter represents the framegrabber type (CommType) and index on which the channel is opened. If only one framegrabber device is to be used, this value should be set to the framegrabber type. Typical supported framegrabber types are: VIRTUAL, EDT_CL, PT_GIGE, MIL_CL and SAPERA_CL. Each framegrabber type is associated with a software plugin which must have been installed correctly in order to use that type of interface. The framegrabber type VIRTUAL allows the use of the SDK without having an operational camera controller available on your system, though the virtual camera it implements simulates only a limited set of the controller's behaviour; once you have installed the plugin providing the virtual camera, please refer to the READ_ME file in the nc_grab_virtual directory. In addition, NC_AUTO_UNIT is an allowed value of this parameter that will automatically detect a unit on which a controller is available. We recommend using it only when a single controller from Nüvü is available to your system (including any VIRTUAL units and operational cameras) since we cannot guarantee the order in which they will be detected. If your system has multiple framegrabber devices of the same type, simply add the device index you wish to use. Note that the device index starts at 0 (e.g. EDT_CL+1 will access the second EDT_CL board) and that the ordering depends upon the exact configuration of your system. However, if you are using PT_GIGE with multiple devices, the order of units is not guaranteed: you may instead want to use ncGrabOpenFromList() after having chosen a device by its serial number (see ncControllerListGetSerial()).
channelThis parameter represents the channel index that is opened. If the first channel is used, this parameter must be set to 0; if the second channel is used, then this parameter should be set to 1; etc. In addition, NC_AUTO_CHANNEL is an allowed value of this parameter that will automatically detect a channel on which a controller is available. We recommend using it only when a single controller from Nüvü is available on the specified unit since we cannot guarantee the order in which they will be detected.
nbrBufferThis parameter represents the number of buffers that will receive acquired images from the controller. These buffers will receive images one after the other such that one or more buffers can be used/processed while another one receives a new image from the controller.
*grabThis parameter is a pointer to the acquisition structure being opened.
Returns
This function returns "0" if successful and an error code if not.

int  timeMs 
)

Sets the time interval at which the camera is pinged by the NIC.

By the end of the mentioned time interval, if the controller still has not answered, the communication port will be closed (which usually means the application must be restarted). In debug mode, when on a breakpoint, the controller won't be pinged unless the executed line is changed. Therefore, when using breakpoints, the heartbeat time should be increased.

Note
This function only applies when using GigE Vision/Ethernet to acquire images.
Parameters
grabThis parameter represents the acquisition structure used to acquire the images. This structure must have been previously defined using the ncGrabOpen() function.
timeMsThis parameter represents the time interval at which the camera is being pinged. The default value is 5000ms. The minimum value that can be set is 2000ms.

int ncGrabFinish ( NcGrab  grab)

Aborts the acquisition currently in progress.

Sets the timestamp mode of the controller.

The timestamp mode specifies if the controller supplies an image timestamp and, if so, how the controller's time is synchronised. The timestamp is the actual time that the controller sends the image to the host (at the exact end of the exposure) and may be retrieved for a specific image using the function ncGrabGetCtrlTimestamp(). To retrieve a valid timestamp, you must first use this function to select a TimestampMode other than NO_TIMESTAMP. The controller's time can be synchronized with a GPS timing module (if connected to the controller and with a valid signal received) by selecting the TimestampMode GPS_TIMESTAMP. The internal controller timestamp can also be synchronised by using the ncGrabSetTimestampInternal function.

To do so, the ncGrabAbort() function can be used to interrupt the acquisition queue and ensure that there are no pending acquisitions. The acquisition queue is emptied (you cannot retrieve the images previously acquired). The counts of missed images, the overrun and timeout flags are reset.
An image timestamp cannot be supplied when using a TDI readout mode. Consequently, neither TimestampMode INTERNAL_TIMESTAMP nor GPS_TIMESTAMP are available with that sort of readout mode.
