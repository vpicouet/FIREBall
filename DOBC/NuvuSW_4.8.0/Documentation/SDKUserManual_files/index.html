Nüvü Camēras SDK Introduction

This SDK allows a user of Nüvü Camēras' devices (cameras or camera controllers) to develop their own software to interface with those devices and use them to acquire and process images. Two types of device control context are provided by the SDK: NcCam and NcGrab. Every application will use one of these two types to interface with a particular device. Each of these types of context is manipulated by a family of functions whose names begin with ncCam or ncGrab, accordingly. Before use, the device control context must be initialised by one of the following functions, depending upon the type of context required: ncCamOpen(), ncCamOpenFromList(), ncGrabOpen() or ncGrabOpenFromList(). The functions ncCamOpenFromList() and ncGrabOpenFromList() make use of a NcCtrlList object which is intended to facilitate selection amongst multiple devices. This object is initialised via either ncControllerListOpen() or ncControllerListOpenBasic() and may be queried via the family of functions whose names begin with ncControllerList.

Most applications should use NcCam because it encapsulates and automates much of the process of configuring the camera controller for readout and the host system for reception of the returned images. In contrast, NcGrab manages principally the reception of images by the host system but virtually none of a camera controller's configuration which must instead be manipulated via the use of serial commands; it should be noted that improper use of certain serial commands may permanently damage the device and/or invalidate your guarantee. For this reason, use of NcGrab should be reserved for cases where the user has the tools (and expertise) necessary to implant customised readout processes on the camera controller, primarily for research and development toward integrating their own camera designs.

For examples of the typical use of NcCam (or NcGrab), please refer to the 'Camera Examples' (or 'Grab Examples') that will be installed alongside the Nüvü Camēras SDK.
