Detailed Description

These functions allow access to the Nüvü Camēras SDK's built-in image processing tools.

These image processing tools assist in the conversion of previously acquired, raw, biased input images into accurate output images, devoid of structure imposed by fluctuations in the bias-level during read out. The tools also permit the construction of photon-counting image stacks, leveraging very high EM gain levels to acheive sub-electron effective redout noise levels.

int ncProcSetProcType ( NcProc  procCtx,
int  procType 
)

This function sets the type of processing to be performed by the context when data images are added to it.

Acceptable processing types are NO_PROC, PC or LM. If the context already contains an image when this function is called, the image already contained is cleared and will replaced by any images that are added (using ncProcAddDataImage).

Parameters
*procCtxThis parameter is a pointer to the processing context which must have been previously defined using the ncProcOpen function. It can be resized using the ncProcResize function.
procTypeThis parameter represents the processing type that will be performed on the images added (using ncProcAddDataImage) to procCtx
  • Use NO_PROC or "0" to get raw images (no processing).
  • Use LM or "1" to get images with bias subtraction.
  • Use PC or "2" to get images in photon counting.
Returns
This function returns "0" if successful and an error code if not.

int ncProcSetOverscanLines ( NcProc  procCtx,
int  overscanLines 
)

This function sets the number of overscan lines for a specified NcProcCtx, which are used to follow the evolution of the noise level from one image to another.

Note
The overscan lines aren't read out from the detector itself but rather are composed of values that are retreived only through the EM registers and the digitizer. Therefore, these lines are close to being completely black. These lines are used to follow the evolution of the intensity offset that could occur from one image to another. Every time one of the ncProcAddDataImage, ncProcProcessDataImageInPlaceForceType or ncProcProcessDataImageInPlace functions is called, the overscanLines are used to update the intensity offset.
Parameters
*procCtxThis parameter is a pointer to the processing context which must have been previously defined using the ncProcOpen function. It can be resized using the ncProcResize function.
overscanLinesThis parameter corresponds to the number of overscan lines that are within the images. The default value is 1.
Returns
This function returns "0" if successful and an error code if not.

int ncProcAddBiasImage ( NcProc  procCtx,
NcImage *  bias 
)

This function adds an image to the processing context in order to create a bias image.

To create a bias image, several images have to be stacked together, meaning that this function should be called recursively. Usually, these images are taken with a closed shutter, using the same camera parameters that will be used to take the images which will be processed using the resulting bias image. We recommend adding at least 50 images to create a meaningful bias image. The more images are added, the more precise the resulting bias image will be. Once all the bias images have been added to the processing context, the bias image can be created and retreived using the ncProcComputeBias function.

Parameters
*procCtxThis parameter is a pointer to the processing context which must have been previously defined using the ncProcOpen function. It can be resized using the ncProcResize function.
biasThis parameter is a pointer to the image that will be added to the processing context in order to create the bias image. This image must be raw data (without any processing applied to it).
Returns
This function returns "0" if successful and an error code if not.
int ncProcComputeBias ( NcProc  procCtx)

This function computes a bias image from all the bias images that were previously added to a specified NcProcCtx.

To add these images, the ncProcAddBiasImage function must be used. Once created, the bias image will be used to process all the images going through the NcProcCtx (using the ncProcAddDataImage, ncProcProcessDataImageInPlaceForceType or ncProcProcessDataImageInPlace function).

Note
This function must be called before attempting to add data images (using the ncProcAddDataImage function).
Parameters
*procCtxThis parameter is a pointer to the processing context which must have been previously defined using the ncProcOpen function. It can be resized using the ncProcResize function.
Returns
This function returns "0" if successful and an error code if not.
int ncProcAddDataImage ( NcProc  procCtx,
NcImage *  imageIn 
)

This function adds an image to a specified NcProcCtx.

Note
This function can only be used if the context is set for PC (photon-counting).

The images added are processed using the bias image previously created (using the ncProcAddBiasImage and ncProcComputeBias functions) and then incorporated into the specified NcProcCtx. To retreive the resulting image, the ncProcGetImage function must be used. The more iterations of ncProcAddDataImage are done, the more depth your resulting image will have.

Parameters
*procCtxThis parameter is a pointer to the processing context which must have been previously defined using the ncProcOpen function. It can be resized using the ncProcResize function.
imageInThis parameter is a pointer to the image that will be processed and added.
Returns
This function returns "0" if successful and an error code if not.

This function returns one processed image from the context.

The returned image is the result of an integration of all the images that have been added to the context using the ncProcAddDataImage function. This function returns an image that is independent from the procCtx and must be freed using the ncProcReleaseImage function.

**imageOutThis parameter is a pointer of pointer to the resulting image after one or multiple iteration of the ncProcAddDataImage function image have been called.
Returns
This function returns "0" if successful and an error code if not.
