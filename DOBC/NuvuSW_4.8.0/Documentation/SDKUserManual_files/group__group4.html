Detailed Description

These functions allow the user to interactively select amongst multiple Nüvü Camēras devices.

The NcCtrlList object is intended to facilitate the management of multiple devices. Once initialised via either ncControllerListOpen() or ncControllerListOpenBasic() it may be queried, in order to identify a particular device, and then used to identify a chosen device to ncCamOpenFromList() or ncGrabOpenFromList() in order to initialise a control context for the chosen device.

Function Documentation

int ncControllerListOpen ( NcCtrlList *  ctrlList)

This function creates a list of controllers that are currently available including their complete model information.

This function tries to read all available CCD and model information from each controller. It is slower than ncControllerListOpenBasic() but will accelerate other controller information fetching functions when compared to lists generated by ncControllerListOpenBasic(). The list generated by this function is required to use other controller listing functions. Controller lists created with this function must be released with ncControllerListFree() when they are no longer needed. Controller lists are static after they are created and cannot be updated. To get an updated list of controllers, a new list must be created.

Note
If you intend to use ncGrabOpenFromList() you should probably use ncControllerListOpenBasic() instead unless you are certain that the camera controller can supply all of the additional information.
Parameters
*ctrlListThe handle pointed to by this parameter will be set to refer to the list that this function creates.
Returns
This function returns "0" if successful and an error code if not.
int ncControllerListOpenBasic ( NcCtrlList *  ctrlList)

This function creates a list of controllers that are currently available.

This function only generates a list of existing controllers without searching for CCD or model information. This information is only requested from a controller when it's explicitly requested by an API call or when a connection to the controller is opened. This function is faster than ncControllerListOpen() unless the intention is to obtain the CCD and model information of each controller in the list. The list generated by this function is required to use other controller listing functions. Controller lists created with this function must be released with ncControllerListFree() when they are no longer needed. Controller lists are static after they are created and cannot be updated. To get an updated list of controllers, a new list must be created.

Parameters
*ctrlListThe handle pointed to by this parameter will be set to refer to the list that this function creates.
Returns
This function returns "0" if successful and an error code if not.

int ncControllerListGetSerial ( const NcCtrlList  ctrlList,
const int  index,
char *const  serial,
const int  serialSize 
)

This function retrieves the serial number of a controller in a list of controllers.

Parameters
ctrlListA handle to a list of controllers. This function will return the serial number of one of the controllers in this list. This handle must be set by ncControllerListOpen() and must not have been released with ncControllerListFree().
indexThe position in the list of the controller whose serial number is requested. Positions in the list are zero based. This parameter must be less than the number of controllers in the list of controllers. The number of controllers that are in a list can be obtained with ncControllerListGetSize().
*serialThe serial number will be written to the buffer pointed to by this pointer. The serial number will be written as a null terminated string of characters. If this parameter is set to null, this function will return the minimum size buffer required to store the serial number, including the string terminator.
serialSizeThe length of the buffer pointed to by the serial parameter. If this parameter is zero or less, this function will return the minimum size buffer required to store the serial number, including the string terminator.
Returns
If serialSize is greater than zero and serial is not a null pointer, this function returns "0" if successful and an error code if not. Otherwise, this function returns the minimum size of buffer required to store the serial number, including the string terminator.

int ncControllerListGetLargestCLinkMode ( const NcCtrlList  ctrlList,
const int  index,
enum CameraLinkMode *  clinkMode,
int *  isSupported,
int *  isAvailable 
)

This function retrieves the largest CameraLink mode required to operate a camera that integrates a controller present in a list of controllers.

A single camera may use different CameraLink modes for readout, where we define the CameraLink mode as the maximum data width in bits that may be sent in a single pixel clock. This function identifies the largest CameraLink mode that may be requested by the camera during its operation. If only the default readout is required, DEFAULT_MODE will be reported. Support for a given CameraLink mode requires the correct framegrabber plugin (see ncControllerListGetPluginName() for more information), and configuration of hardware/cables, so this function also reports both if the reported CameraLink mode is supported by the plugin and that mode is available in the current configuration.

Parameters
ctrlListA handle to a list of controllers. This function will return the detector type of the camera connected to one of the controllers in this list. This handle must be set by ncControllerListOpen() and must not have been released with ncControllerListFree().
indexThe position in the list of the controller whose detector type is requested. Positions in the list are zero based. This parameter must be less than the number of controllers in the list of controllers. The number of controllers that are in a list can be obtained with ncControllerListGetSize().
*clinkModeThe largest CameraLink mode required to operate the camera will be written to the buffer pointed to by this pointer.
*isSupportedThe status of the plugin's support for the reported CameraLink mode will be written to the buffer pointed to by this pointer. It will be one (1) if the mode is supported and zero (0) if it is not.
*isAvailableThe availability of the reported CameraLink mode in the current configuration of the framegrabber/cables will be written to the buffer pointed to by this pointer. It will be one (1) if the mode is supported and zero (0) if it is not.
Returns
This function returns "0" if successful and an error code if not.
