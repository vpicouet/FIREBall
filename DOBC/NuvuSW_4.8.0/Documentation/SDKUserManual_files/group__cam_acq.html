Detailed Description

These functions allow the acquisition of images via a Nüvü Camēras device (i.e. the acquisition, processing and saving of images read out from the device) via a NcCam handle.

Devices provided by Nüvü Camēras with pre-programmed readout modes, such as the HNü or EMN2 models of camera, will support the use of NcCam as the control context and it is recommended for most use cases.

Enumeration Type Documentation

enum ProcType

Values identifying some available image processing mode settings.

This enum is for legacy compatibility, however the values are suitable for use with the functions ncCamSetProcType() and ncGrabSetProcType().

Enumerator
NO_PROC 

No processing; raw images.

LM 

Linear or analogue mode; bias subtraction of images to a flat consistent baseline (bias clamp)

PC 

Photon-counting mode; bias subtraction and thresholding to eliminate ENF (requires high EM-gain and low light levels).

enum NcSpecialMode

Values identify available special operating modes.

For use with the ncCamSetSpecialMode() function.

Enumerator
NC_SPECIAL_MODE_NORMAL 

Normal operation mode, images are received and processed as normal.

NC_SPECIAL_MODE_SERIAL_ONLY 

Serial communication only, custom restrictions are in effect, no images are processed.

enum NcByteLayoutFormat

List of byte layout format types.

For use with the ncGrabSetReadoutByteLayoutStr function.

Enumerator
NC_BYTE_LAYOUT_FORMAT_FLAT_LAYOUT 

Manually define each byte's position in the byte layout.

int ncCamStart ( NcCam  cam,

Starts image acquisition.

The host system will wait for the requested number of images from the camera. However, this function will return immediately; it will not wait for the images to be acquired.

Note
To launch a continuous acquisition, set "nbrImages" to 0.

Returns
This function returns "0" if successful and an error code if not.

This sets up a callback function that will be raised whenever an acquisition's processing pipeline fully completes and all images have been pushed to the read queue. ncCamAbort, ncCamFinish and simply completing a fixed length acquisition will raise this callback. This callback should be set before stopping the acquisition or, in the case of a fixed length acquisition, before the acquisition is started. Setting the callback any later may result in a race condition where the event occur before setting the callback completes causing the event to be lost.

Setting a callback will replace any previously set callback. The first argument given to the callback is the same NcCam context used to call this function. The second argument is a copy of the data parameter.

This function estimates the parameters of the fit of the Gaussian curve over the vertical and horizontal sum of a region.

Note
The region must be entirely contained inside the image area, otherwise this function will fail.
Diagonal gaussians might not be properly calculated. For optimal result, the gaussian axis should be horizontal or vertical.
Parameters
camThis parameter represents the camera structure used to acquire the images. This structure must have been previously defined using the ncCamOpen function.
regionIndexThis parameter is the index of a region within the camera structure, which must have been previously defined using the ncCamStatsAddRegion function. This region will be resized according to the regionWidth and regionHeight specified.
*imageThis parameter is a pointer to the image on which the statistics will be calculated.
xCoordThis parameter defines the x coordinate for the start of the region in which the statistics will be calculated, beginning from the lower left corner of the acquired image.
yCoordThis parameter defines the y coordinate for the start of the region in which the statistics will be calculated, beginning from the lower left corner of the acquired image.
*maxAmplitudeThis parameter returns the maximum amplitude found by the correlation of the two gaussians (one on each axis) calculated by this function.
gaussSumHorizontalThis parameter returns the values of the estimated Gaussian fit performed over the horizontal sum of the region.
gaussSumHorizontal[0] represents the amplitude;
gaussSumHorizontal[1] represents the center;
gaussSumHorizontal[2] represents the FWHM.
gaussSumVerticalThis parameter returns the values of the estimated Gaussian fit performed over the vertical sum of the region.
gaussSumVertical[0] represents the amplitude;
gaussSumVertical[1] represents the center;
gaussSumVertical[2] represents the FWHM.
useActualCrossSectionFlagThis parameter indicates wheter or not to use the current cross section associated with the regionIndex specified in the grab structure. This should be used if the ncGrabStatsGetHistoCrossSection function was previously used on the same image with the same regionIndex, image, xCoord and yCoord parameters.
Returns
This function returns "0" if successful and an error code if not.
